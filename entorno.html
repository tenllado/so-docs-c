3 Compilacin de proyectos en Linux

En programacin consideramos un proyecto el conjunto de ficheros necesarios para producir una aplicacin. Para cada aplicacin emplearemos un directorio donde se alojarn todos los ficheros relacionados con ella, que bsicamente son de 6 tipos:

  1. Uno o ms ficheros fuente escritos en lenguaje C (con extensin .c)

  2. Uno ms ficheros objeto generados por compilacin de cada fichero fuente (con extensin .o)

  3. Uno o ms ficheros de cabecera referenciados mediante sentencias #include en los ficheros fuente (con extensin .h)

  4. Un fichero de nombre Makefile que contendr las reglas de construccin de la aplicacin y que la utilidad make consulta, habitualmente para generar el ejecutable final.

  5. Opcionalmente, uno o ms ficheros de biblioteca que contendrn cdigo objeto

  6. Un fichero ejecutable

3.1 Compilador GCC

Para compilar los ficheros fuente utilizaremos el compilador de C de GNU, gcc. En realidad gcc es una herramienta que sirve de interfaz comn para el uso de distintas herramientas del proceso de compilacin, como son el preprocesador, el compilador propiamente dicho (cc), el ensamblador (as) y el enlazador (ld). El proceso que normalmente llamamos compilacin es una secuencia de los siguientes pasos:

  1. Un preprocesador (cpp), que toma como entrada un fichero fuente archi.c y lo transforma en un fichero intermedio archi.i mediante la realizacin de las operaciones de pre-proceso textual demandadas por las instrucciones \#xxx que aparecen en el cdigo fuente, denominadas macros del preprocesador. Las principales macros son #include, #ifdef, #ifndef, #else, #endif, #define,... (Consultar un tutorial de Lenguaje C para conocer el significado de tales instrucciones).

  2. Un compilador de C (cc) que toma como entrada el fichero intermedio archi.i y produce cdigo textual en lenguaje ensamblador propio de la arquetectura destino, que almacena en un fichero archi.s.

  3. Un ensamblador (as), que toma como entrada el fichero archi.s y genera cdigo mquina que almacena en un fichero con formato ELF reubicable de nombre archi.o.

  4. Un enlazador (ld) que monta uno o varios ficheros .o, resuelve las referencias cruzadas entre ellos y con bibliotecas de cdigo objeto, especialmente la biblioteca bsica de C de nombre libc.a (versin de enlace esttico) o libc.so (versin de enlace dinmico). Si no queda ninguna referencia por resolver, el enlazador genera un fichero ELF ejecutable, de nombre a.out por defecto, utilizable como punto de partida para crear una imagen de proceso.

La herramienta gcc admite una serie de opciones al ser invocada, las ms usadas se muestran en la siguiente tabla:

Opcin Explicacin
-c Realiza slo los tres primeros pasos: preproceso, compilacin y ensamblaje
-g Durante la compilacin crea una tabla de smbolos (asociacin entre nombres de funciones o variables y direcciones de referencia) que almacena en los ficheros objeto y ejecutable generados y que puede ser consultada por una utilidad de depuracin para permitir depuracin simblica
-Ox Demanda aplicar distintos niveles de optimacin durante el proceso de compilacin y generacin de cdigo mquina. Los niveles ms usuales son s (optimizacin en tamao), 0 (anular optimizacin), 1, 2 y 3 (distintos grados cada vez ms exigentes de optimizacin)
-o nombre Propone nombre como nombre del fichero de salida resultante en vez del nombre utilizado por defecto (que, en caso del ejecutable final, es a.out)
-I directorio Instruye al preprocesador para que busque los ficheros de cabecera referenciados en el directorio designado adems de los directorios por defecto /include y /usr/include
-D xxx Define una variable de preproceso de nombre xxx. Tambin se le puede dar un valor si se utiliza -D xxx=valor
-L directorio Instruye al enlazador a que busque las bibliotecas propuestas, en el directorio designado adems de los directorios por defecto /lib y /usr/lib
-lxxx Instruye al enlazador para que utilice la biblioteca de nombre libxxx.a (versin esttica) o libxxx.so (versin dinmica) existente en algunos de los directorios especificados implcita o explcitamente
-static o -dynamic Escoge entre enlace con bibliotecas estticas o dinmicas (si no se indica nada, se aplica enlace dinmico)

-Wall Habilita los avisos del compilador para mltiples errores comunes, es muy recomendable usarlo siempre

Como ya dijimos, la primera etapa de compilacin es el preprocesado. El preprocesador examina los ficheros .c en busca de declaraciones que comiencen por #; cuando son del tipo #include localiza el fichero designado y lo aade al programa. A continuacin el preprocesador busca y sustituye todas las macros (#define) y realiza las sustituciones de texto indicadas; el compilador propiamente dicho (etapa segunda), no compila el fichero .c original sino la suma del .c y .h con las macros sustituidas. Esto puede producir algunos problemas serios si el fichero de cabecera contiene errores. Cuando se emplean macros que se comportan como funciones, y existe algn error en la macro, puede resultar difcil detectar la causa del error; el compilador no sabe de cabeceras, y cuando ve un error, informa acerca del nmero de lnea que l ve. Al buscar tal lnea en el fichero .c original puede que nos encontremos una sentencia aparentemente inocente.

3.2 GNU Make

Es frecuente durante el desarrollo del proyecto que haya que compilar dicho proyecto multitud de veces. Adems, el proyecto va cambiando, aadindose ficheros, bibliotecas con las que se debe enlazar, ejecutables y bibliotecas que se deben generar, etc. Repetir en cada ocasin las llamadas a gcc para compilar cada objetivo del proyecto es ineficiente y propenso a errores. La popular herramienta Make, diseada para entornos UNIX, tiene como objetivo facilitar y, en cierta medida, automatizar este proceso de compilacin del proyecto.

Hay distintas versiones de la herramienta Make, sin embargo todas ellas funcionan de un modo muy parecido. Cuando se invoca, Make interpreta un fichero de texto con una sintaxis especial y nombre preestablecido (p.ej. makefile), que le indica los pasos que debe seguir para compilar el proyecto. Una vez confeccionado el makefile repetir la compilacin es tan sencillo como ejecutar make desde el terminal.

En esta asignatura emplearemos la implementacin de Make de GNU, que se encuentra instalada en el laboratorio. Cuando el usuario invoca el comando make desde el shell, la herramienta busca en primer lugar un fichero que se llama GNUmakefile. Si no se encuentra se busca un fichero llamado makefile, y si por ltimo no se encontrase, se buscara el fichero Makefile. Si no se encuentra en el directorio actual ninguno de esos tres ficheros, se producir un error y make no continuar:

$ make 
make: *** No targets specified and no makefile found.  Stop.

En caso contrario, make procesar el archivo de texto e invocar el conjunto mnimo de reglas requeridas para construir nuestro proyecto. Para ilustrar el flujo de trabajo de make, consideremos el siguiente ejemplo. Supongamos que tenemos un proyecto C que consta de un Makefile correctamente construido y dos archivos .c, que deben compilarse y enlazarse para construir el ejecutable de un programa. Normalmente, las reglas del fichero Makefile garantizan que si solo se ha modificado un archivo C desde la ltima vez que se invoc make por ltima vez, una nueva invocacin de make realizar la compilacin solo para el archivo C modificado y la etapa final de enlazado. Por tanto, el fichero objeto intermedio asociado al archivo C sin modificar no se volver a generar.

En este curso, la mayor parte de los proyectos C de ejemplo proporcionados con las distinas prcticas se acompaan de un fichero Makefile especfico para el proyecto. Por lo general, para compilar el proyecto asociado bastar teclear make en la terminal. Adems, la mayora de los Makefiles contarn tambin con una regla de limpieza para eliminar todos los archivos intermedios generados durante el proceso de compilacin. En particular, al ejecutar el comando make clean desde el directorio principal del proyecto, los archivos de objeto y ejecutables correspondiente se eliminarn automticamente.

Cabe destacar que durante el desarrollo de la asignatura los estudiantes no tendrn que elaborar ficheros Makefile de cero, sino adaptar mnimamente ejemplos ya proporcionados. Por lo tanto en este documento no procederemos a introducir la sintaxis de GNU make. No obstante, esta sintaxis est descrita en el tutorial disponible en este enlace.

4.2 Depuracin de un programa de ejemplo

Procedamos ahora a utilizar gdb para depurar un programa que se ejecuta incorrectamente, y as poner de manifiesto las principales facilidades de que dispone.

El programa se llama badsort.c y realiza una ordenacin por el mtodo de la burbuja aplicada a un array de datos inicializado dentro del programa. El cdigo se suministra como material suplementario.

Un aviso (warning) del compilador nos ayudar a detectar el primer error. Le pedimos a gcc que nos informe de cualquier circunstancia sospechosa del cdigo an cuando no sea necesariamente un error fatal. Para ello utilizaremos la opcin -Wall. Tambin emplearemos la opcin -g para permitir la depuracin simblica del programa en ejecucin. Escribimos pues:

badsort.c: In function 'sort':
badsort.c:20:17: warning: suggest parentheses around comparison 
in operand of '&' [-Wparentheses]

En el mensaje de compilacin nos llaman la atencin sobre la lnea 20 que contiene
/* 20 */ for(; i < n & s != 0; i++) {

Una descripcin completa de la biblioteca queda fuera del alcance de este documento. 6.1 Gestin del Heap

En el mapa de memoria de un proceso, suele reservarse una zona de memoria conocida como Heap. Esta zona de memoria se utiliza para que el programador pueda ir solicitando trozos de distinto tamao dinmicamente, que suele utilizar para alojar variables creadas en tiempo de ejecucin. Es decir, en lugar de reservar espacio para una variable global en tiempo de compilacin, o reservarla en la pila como variable local de alguna funcin, el programador solicita al sistema la memoria que necesita para alojar la variable, y el sistema le reserva un trozo del heap.

En C la gestin del Heap se realiza principalmente a travs de dos funciones: malloc, para solicitar memoria, y free, para liberarla. Las cabeceras de estas funciones se declaran en el fichero stdlib.h, y son:

  • void* malloc( size_t size );

    La funcin malloc toma como parmetro el nmero de bytes que se desea reservar, y devuelve un puntero con la direccin de memoria del bloque reservado, o NULL en caso de que no pudiese reservar esta memoria (por ejemplo, que no quede espacio en el heap).

  • void free( void* ptr );

    La funcin free toma una direccin previamente devuelta por malloc y la vuelve a aadir como memoria libre/disponible al heap. Es importante que no se intente liberar con free memoria a partir de una direccin no devuelta por malloc, esto provocara excepcin, haciendo que el programa finalice abruptamente con error.

Existen otras funciones para la gestin del Heap, como calloc, realloc, valloc y reallocf. Se recomienda al alumno consultar sus pginas de manual.

6.2 Funciones de E/S estndar

Las principales funciones para escribir sobre el terminal o leer lo que introduce el usuario desde el teclado son: printf y scanf. Las cabeceras de estas funciones se declaran en el fichero stdio.h, y son:

  • int printf(const char * restrict format, ...);

    La funcin printf permite mostrar por la salida estndar el valor de cualquier variable de tipo nativo (int, char, etc), con un formato especificado por una cadena de caracteres. La cadena de caracteres de formato puede incluir cero o ms directivas, caracteres normales distintos de % que son escritos tal cual en el terminal, y especificadores de conversin. Cada especificacin de conversin comienza con el carcter %, seguido de uno o ms caracteres que describen la conversin. Su funcin es obtener un texto que describa el contenido de una variable nativa, que ser insertado en esa misma posicin en el flujo de salida (terminal).

    Por ejemplo, supongamos que tenemos una variable entera n, y queremos mostrar por terminal la cadena de texto: n = valor_de_n. Usaramos la funcin printf de la siguiente manera:

    printf("n = %d", n );

    Tambin podemos aadir a la cadena de formato secuencias de escape, que comienzan por el carcter \, seguido de otro carcter. Estas secuencias sirven para incluir, por ejemplo, saltos de lnea (\n).

    El formato concreto de todos los especificadores de conversin y las secuencias de escape puede encontrarse consultando la pgina de manual de printf o consultando cualquier manual de C.

  • int scanf(const char *restrict format, ...);

    La funcin scanf sirve para leer de la entrada estndar. Al igual que printf toma una cadena de caracteres para especificar el formato de lo que se espera leer de la entrada estndar, que incluye especificadores de conversin. Cada especificador de conversin se corresponde con una variable que debe ser pasada a scanf por referencia (pasamos la direccin de la variable), de forma que al hacer la conversin de su valor en texto a su valor numrico el resultado se almacenar en dicha variable.

    Siguiendo con el ejemplo, si esperamos leer una cadena como la del caso anterior, y asignar el valor a la variable n, escribiramos:

    scanf("n = %d", &n );

    La funcin devuelve el nmero de asignaciones realizadas (conversiones). Se recomienda al alumno consultar su pgina de manual para completar esta descripcin.

La biblioteca estndar de C ofrece una serie de funciones que permiten el acceso a ficheros. Las cabeceras de estas funciones se declaran en el fichero stdio.h. En esta seccin vamos a describir slo algunas de las funciones ms importantes, que son:

  • FILE * fopen(const char *restrict filename, const char *restrict mode);

    La funcin fopen abre el fichero de nombre indicado por la cadena de caracteres que se pasa como primer argumento. El nombre puede ser una ruta absoluta o relativa. El segundo argumento de la funcin es una cadena de caracteres que indicar el modo en que debe abrirse el fichero:

    Si tiene xito, la funcin devuelve la direccin de una estructura FILE que se usa como descriptor del fichero, y deber pasarse al resto de funciones para trabajar con dicho fichero. Si se produce algn error, fopen devolver NULL.

  • int fseek(FILE *stream, long offset, int whence);

    La funcin fseek permite desplazar el indicador de posicin del fichero asociado al descriptor cuya direccin se pasa como primer argumento. La nueva posicin, medida en bytes, se obtiene sumando offset bytes a la posicin indicada por el tercer argumento whence. ste ltimo puede tomar tres posibles valores:

    SEEK_SET

    comienzo del fichero

    SEEK_CUR

    posicin actual

    SEEK_END

    final del fichero

    En caso de xito, fseek devuelve 0. En caso de error, la funcin devuelve -1 y asigna a la variable global errno un valor que describe el motivo del error.

  • int putc(int c, FILE *stream);

    La funcin putc escribe el valor entero c convertido a unsigned char en el fichero asociado al descriptor cuya direccin se pasa como segundo argumento. Si tiene xito devuelve el carcter escrito. En caso de error devuelve EOF.

  • int getc(FILE *stream);

    La funcin getc lee un carcter (1 byte) del fichero asociado al descriptor cuya direccin se pasa como argumento. Si tiene xito devuelve el carcter ledo como unsigned char transformado a entero. En caso de error devuelve EOF.

  • int feof(FILE *stream);

    La funcin feof comprueba si se ha activado el indicador de final de fichero en el descriptor cuya direccin se pasa como argumento, devolviendo un valor no nulo si est activo. Este indicador se puede borrar explcitamente invocando la funcin clearerr() o como efecto lateral del uso de fseek().

  • char* fgets(char *str, int size, FILE *stream);

    La funcin fgets lee como mucho size-1 caracteres del fichero cuyo descriptor se pasa como parmetro (stream), y los almacena en el buffer apuntado por str. La lectura finaliza cuando se llega a fin de fichero o se procesa un salto de lnea. En caso de que se detecte un salto de lnea, ste se almacena en el buffer, que en cualquier caso incluye el caracter terminador al final.

  • int fprintf(FILE  *stream, const char *format, ...)

    Se trata de una variante de la funcin printf donde el mensaje con formato se imprime en el fichero cuyo descriptor se pasa como primer parmetro (stream).

6.4 Comprobacin de errores

Como hemos visto en las secciones anteriores, muchas funciones de la biblioteca estndar de C asignan un valor determinado a la variable global errno para indicar la causa por la que han finalizado con error. La biblioteca ofrece tambin una serie de funciones que permiten interpretar estos valores y comunicarlos al usuario de forma ms amigable. Una de las funciones ms utilizadas para ello es:

  • void perror(const char *s);
    La funcin perror muestra por pantalla una cadena de caracteres terminada en final de lnea, con un mensaje que describe el error indicado en ese momento por la variable errno. Adems, acepta como argumento la direccin de una cadena de caracteres. Si ste argumento no es NULL, dicha cadena se pone delante del mensaje descriptivo, separando ambos por el carcter :.
